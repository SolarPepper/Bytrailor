## Security Best Practices

### API Keys
- ✅ Храните API ключи только в файле `.env`
- ✅ Файл `.env` добавлен в `.gitignore` и не попадает в репозиторий
- ✅ Используйте `.env.example` как шаблон
- ⚠️ Никогда не коммитьте реальные API ключи

### Docker Security
- ✅ Контейнер запускается от non-root пользователя
- ✅ Минимальные права доступа (cap_drop: ALL)
- ✅ Read-only файловая система
- ✅ Ограничения ресурсов (CPU/Memory)
- ✅ Регулярное сканирование на уязвимости через CI/CD

### Runtime Security
- ✅ Используется только HTTPS/WSS для подключения к Bybit
- ✅ Проверка входных данных (safe_float функции)
- ✅ Обработка исключений
- ✅ Логирование для аудита

## Reporting a Vulnerability

Если вы обнаружили уязвимость безопасности, пожалуйста:

1. **НЕ создавайте публичный Issue**
2. Отправьте email на: [ваш email] или создайте Security Advisory через GitHub
3. Опишите уязвимость детально
4. Укажите шаги для воспроизведения (если возможно)

Мы ответим в течение 48 часов и предоставим обновление, когда проблема будет решена.

## Automated Security Checks

Проект использует следующие инструменты для автоматической проверки безопасности:

- **Bandit**: Статический анализатор безопасности Python кода
- **Safety**: Проверка уязвимостей в зависимостях
- **pip-audit**: Аудит Python пакетов
- **Trivy**: Сканирование Docker образов на уязвимости
- **Docker Scout**: Анализ безопасности Docker образов
- **Hadolint**: Линтинг Dockerfile на best practices

Все проверки запускаются автоматически через GitHub Actions при каждом push.
